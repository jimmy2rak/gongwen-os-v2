- 第一次走 POST 不触发，第二次走 PUT 触发更新就崩
- 修复：删除 updateData.data

## Bug 修复（2026-07-03 下午）

### Bug 1: 保存文档失败
- 原因：`db.batch()` 在部分 drizzle-orm/libsql 版本有兼容问题
- 修复：拆分为两次独立 `db.insert()` 调用
- 新增：API 返回完整 error message 到前端
- 新增：保存成功后自动更新浏览器 URL 为 `/documents/[id]`
- 新增：页面加载时如果 URL 包含文档 ID，从数据库加载内容

### Bug 2: 红头/文号/印章按钮无反应
- 原因1：扩展定义了 `content: "inline*"` 但 toolbar 传入 `content: [{ type: "paragraph" }]`（block），ProseMirror 静默拒绝
- 原因2：Seal 按钮用了 `(editor as any).chain().focus().setSeal()` 尝试调用不存在的 TipTap 命令
- 修复1：三个扩展改为 `atom: true` 原子节点，`content: ""` 无子节点，简化渲染
- 修复2：统一使用 `cmd(editor).setSeal(text)` 辅助函数，不走 TipTap chain API

## 修复 Router 渲染错误
- 问题：`router.push("/login")` 在渲染阶段直接调用，React 报错 "Cannot update a component while rendering another"
- 涉及文件：`(dashboard)/layout.tsx`、`page.tsx`、`(dashboard)/documents/page.tsx`
- 修复：把 `router.push` 从渲染阶段移到 `useEffect` 中
- 布局文件：`(dashboard)/layout.tsx` → 所有 dashboard 子页面自动受保护（文档页不再需要单独写 auth check）
- 首页：`page.tsx` → 根路由自己的 auth check 也移到 useEffect

## 修复无限轮询 + 渲染冲突
- **根因1**：`auth.store.ts` 的 `fetchUser()` 每次调用都设 `isLoading=true`，再设回 `false`，导致所有依赖 `isLoading` 的组件反复重渲染
- **根因2**：文档页 `useEffect(() => fetchUser(), [])` 和布局组件的 `fetchUser()` 双重调用。文档页的 fetchUser 设 isLoading=true → 布局变 loading 卸载子页面 → 文档页卸载 → fetchUser 完成 isLoading=false → 布局加载子页面 → 文档页又挂载 → fetchUser 又调了 → ♾️ 死循环
- **修复**：
  1. `auth.store.ts`：`fetchUser()` 不再无脑设 `isLoading=true`，只在初始态保持 true，API 完成后设 false。删除 `set({ isLoading: true })`
  2. `(dashboard)/layout.tsx`：`fetchUser()` 加 `useRef` 确保只执行一次；`router.push` → `router.replace`
  3. `(dashboard)/documents/page.tsx`：删除自己的 `fetchUser()` 调用，由布局统一管理；删除 `isLoading` 引用
  4. `page.tsx`：同样加 `useRef` 防重复调用；`router.push` → `router.replace`
