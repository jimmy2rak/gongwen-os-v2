## 一键初稿 Markdown 渲染修复 + Skill 树状选择器

### 问题
1. 一键初稿/大纲生成的内容追加到编辑器后，Markdown 语法（`###`、`####`、`**`、列表等）残留，没有正确渲染为 TipTap 节点。
2. 一键初稿、大纲生成页面没有 Skill 选择器，用户无法指定生成时引用哪些写作规范。

### 解决方案

1. **引入 `marked` 库**
   - `npm install marked`
   - 新建 `src/lib/markdown.ts`：
     - `looksLikeMarkdown()`：检测文本是否含 Markdown 特征
     - `markdownToGovDocHtml()`：用 marked 解析，并把第一个标题（优先 h1，否则 h2/h3/h4）映射为 `<div data-type="doc-title">`，其余标题保持层级
     - `safeTextToGovDocHtml()`：纯文本兜底

2. **修复编辑器渲染链路**
   - `src/components/editor/DocEditor.tsx`：`templateMarkdownToHtml` 改为调用 `markdownToGovDocHtml`，内容 Markdown 检测改用 `looksLikeMarkdown`
   - `src/lib/doc-create.ts`：保存为文档前识别 Markdown 并转换
   - `src/app/page.tsx`：一键初稿追加/替换到编辑器时，先转换 Markdown 为 HTML 再写入 store
   - `src/app/(dashboard)/quick-draft/quick/page.tsx` 与 `outline/page.tsx`：生成结果预览区改用 HTML 渲染（`gov-doc-preview` 样式）

3. **新增 Skill 树状选择器**
   - 新建 `src/components/quick-draft/SkillSelector.tsx`：复用 AI chatbox 的树状选择逻辑，支持全局 Skill + DB 分类 Skill 多选
   - `src/lib/ai/use-generate.ts`：`run()` 增加 `extraSystemExtra` 参数
   - `src/lib/ai/stream-client.ts`：`StreamChatOptions` 增加 `extraSystemExtra`；提供额外上下文时跳过自动全量分类 Skill，避免重复
   - 在「出稿」和「大纲」页左侧面板加入 `<SkillSelector />`，选择的 Skill 上下文真实注入 AI system prompt

4. **优化 Prompt**
   - 出稿/大纲 prompt 追加 Markdown 输出格式要求：`#` 主标题、`##/###/####` 小标题、段落空行、列表用 `1.`/`-`
