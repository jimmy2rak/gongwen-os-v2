## 修复 HomePage Hooks 顺序错误（2026-07-07 01:48）
- **报错**：React "change in the order of Hooks called by HomePage"，上一帧 25 hook、下一帧 26（多一个 useEffect），导致 500 类崩溃。
- **根因**：`src/app/page.tsx` 在顶部有加载守卫 `if (isLoading || !user) return <加载中>`，但拉取模板列表的 `useEffect([store.showNewDocDialog])` 写在了这个 early return **之后**（原 line 231）。加载态时 early return 触发 → 跳过该 useEffect（25 个）；加载完成后不 return → 多执行一个（26 个）。违反 Rules of Hooks。
- **修复**：把该 `useEffect` 整体移到 early return **之前**（紧贴 URL-load useEffect 之后），所有 hook 无条件执行；删除原位置的重复块。脚本校验"early return 之后无 hook"通过；`tsc --noEmit` 0 错误。
- **教训**：任何 early return / 条件 return 必须放在组件所有 hook 之后。本次是 P5 模板接入时新增的 useEffect 放错位置。
